# 밤바다 · 迷宮 — 설계 캔온 (DESIGN.md)

> 이 문서가 단일 진실원본(canon)이다. 결정은 채팅이 아니라 여기에 적는다.
> 코드 = `index.html`(현재 단일 파일). 라이브 = https://elpalaiso.github.io/mobilemaze/

## 1. 한 줄
휴대폰으로 푸는 **미궁(수수께끼) 게임**. 20여 년 전 PC 미궁의 향수를, *폰에서만 되는 트릭*으로 더 세련되고 다채롭게.

## 2. 핵심 원칙
- **트릭은 스토리에 녹인다 (diegetic).** 제스처가 곧 이야기 속 행동이다. "기믹 모음"이 아니라, *그 동작이 항해의 한 단계.*
- **모바일 전용 트릭이 차별점.** 길게누르기·핀치·기울이기·불기·흔들기… 데스크탑/소스보기 꼼수가 안 통하는 게 매력.
- **향수 + 세련.** 옛 미궁의 수수께끼 감성은 살리고, 손맛은 폰 네이티브로.
- **선한 흐름.** 어둡지 않게, 아련하고 포근한 밤바다 톤.

## 3. 현재 상태 (2026-06-19)
- 구조: **튜토리얼 3 (연습) + 출항 1 (첫 항해) + 클리어**.
  - 연습1 *길게누르기* → `달빛`
  - 연습2 *핀치줌*(3px 깨알) → `등대`
  - 연습3 *기울이기*(gamma>40°) → `수평선`
  - 출항 *불기*(마이크 소리로 돛 채우기, 100%면 출항) — 제스처 완료형(타이핑 없음), 마이크 불가 시 "노 저어 출발" 폴백
- 시스템: **진행저장**(localStorage `mobilemaze.progress`) · **다시하기**(UI 상태까지 초기화) · **i18n**(한/영, 자동감지+토글, `mobilemaze.lang`).

## 4. 트릭 어휘 (= 항해 행동 매핑)
**구현됨**
| 트릭 | 제스처 | 항해 의미 | 비고 |
|---|---|---|---|
| 길게누르기 | 600ms 꾹 | 등불 켜기 | touch/mouse |
| 핀치줌 | 두 손가락 확대 | 깨알 단서 읽기 | 3px 텍스트 |
| 기울이기 | gamma>40° | 키 잡고 수면 맞추기 | iOS 권한버튼 |
| 불기 | 마이크 입김 | 돛 펴고 출항 | getUserMedia, 폴백 有 |

**후보 풀 (다음 발굴)**
- 흔들기(devicemotion) = 닻 올리기 / 안개 걷기 *(iOS 권한 필요)*
- 손가락으로 긋기(canvas) = 항로 그리기 *(권한 무, 안정적)*
- 화면 회전(두 손가락) = 나침반 돌리기
- 마이크 음량 지속 = 뱃노래 / 외침
- 화면 밝기·배터리·시간 = 환경 단서
> 새 트릭은 **각자 `init/cleanup`을 갖는 모듈**로(레지스트리 패턴). iOS 권한·HTTPS 필요 여부를 트릭마다 명시.

## 5. 스토리 프레임 — "밤바다 건너기"
- 큰 틀: 플레이어가 **밤바다를 건너는 항해.** 각 챕터 = 항해의 한 구간/섬/등대.
- 튜토리얼(연습 3) = 출항 전 *바다를 읽는 법* 배우기 → 동작 어휘 습득.
- 이후 챕터 = 어휘를 *조합*해 진짜 수수께끼. 트릭이 곧 항해 사건(돛·키·닻·항로…).
- 톤: 아련·포근, 등불과 파도. 막 끝마다 작은 도착(섬/등대)의 안도.

## 6. 레벨 설계 규칙
- 튜토리얼은 트릭 1개씩 *가르치고*, 본편은 *조합/응용*.
- **폴백 필수**: 권한 거부·미지원(특히 인앱 브라우저)일 때 막히지 않게 우회 경로.
- 정답은 **개념 기반**(달빛·등대·수평선)으로 — 언어가 달라도 번역되게(말장난 의존 최소화).
- 한 챕터 = 짧게. 도착의 보상감 > 길이.

## 7. i18n 정책
- 모든 텍스트·정답은 `I18N` 사전(`ko`/`en`)에 키로. 새 레벨 = 사전에 키만 추가.
- **트릭형 퍼즐** = 텍스트만 번역하면 다국어 OK(메커니즘 언어무관).
- **말장난형 퍼즐** = 로케일별로 *퍼즐 자체를 재설계* (기계번역 금지). 정답도 언어별.
- 자동감지(`navigator.language`) + 수동 토글, 선택 저장.

## 8. 기술 아키텍처
- **현재: 단일 `index.html`** (자기완결 HTML+CSS+JS, 의존성·빌드 0). 지금 규모엔 최적.
- **분기점: 트릭 2~3개째 추가 시 분리** →
  ```
  index.html   (껍데기)
  styles.css   (스타일)
  engine.js    (레벨 시스템·i18n·진행저장·트릭 레지스트리)
  levels.js    (퍼즐 데이터 + 문구 — 콘텐츠만)
  ```
  → **새 레벨 = `levels.js` 데이터만 추가**, 엔진 불변. 데이터/로직 분리.
- 더 커지면(수십 레벨·에디터): Vite 등 빌드툴(결과는 여전히 정적 → Pages 무료).
- **쪼개도 무료 배포 유지.** 정적 파일은 개수 무관.

## 9. 배포 운영
- **GitHub Pages**, repo `github.com/elpalaiso/mobilemaze`(public), URL 위.
- push: gpt-5.5 서브에이전트 위임 가능하나 **프록시 `127.0.0.1:443` 간헐 실패** → 3회 재시도 실패 시 **오너 터미널서 `git push`**(커밋은 로컬 안전).
- gh CLI는 **오너 터미널에서만** 작동(서브에이전트는 keyring 못 읽음). 자동화 필요 시 `GH_TOKEN` 환경변수.
- 첫 배포/변경 후 Pages 반영 ~1분, 초기 404 정상.
